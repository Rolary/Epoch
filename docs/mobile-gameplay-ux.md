# 《生态纪元》手游操作与引导设计

> UI 视觉原型已沉淀到 [mobile-ui-prototypes.md](./mobile-ui-prototypes.md)。后续前端页面优先参考这些原型的布局、信息层级和游戏感。

## 1. 设计目标

《生态纪元》的手机端体验应像一款可玩的放置手游，而不是一个资源控制面板。

玩家打开游戏后的第一感受应是：

```text
我正在照看一个活着的始源潮池。
我知道下一步要做什么。
我每次点击都会让潮池发生一点变化。
新系统会随着生命史推进逐步出现。
```

## 2. 核心原则

- 首屏只放一个主场景、一个当前目标、一个主要操作。
- 玩家不能在开局看到完整系统菜单。
- 资源、环境、演化树、图鉴、日志都应被包装成游戏进程的一部分，而不是管理后台。
- 每次新功能出现，都需要由目标卡、事件反馈或剧情提示承接。
- 任何新解锁功能第一次出现时都必须有短引导，包括第一次进入游戏。
- 主操作必须开局可用，不能因为资源不足让玩家第一下就失败。
- 复杂数值必须折叠为进度、状态或简短标签。
- UI 控件优先使用“图标 + 小字说明”，减少大段文字按钮。

## 3. 首屏结构

开局首屏只包含：

```text
当前目标
始源潮池主场景
主操作按钮
关键进度条
最近反馈
已解锁的少量入口
```

不应出现：

- 完整资源表
- 完整环境参数表
- 全量演化树
- 全量底部导航
- 多个并列系统面板

## 3.1 创建生态

第一次进入游戏时，不直接创建匿名存档。

创建流程：

```text
输入生态名称
-> 展示 3 个随机源质印记
-> 选择 1 个作为开局永久天赋
-> 创建生态
-> 进入始源潮池
```

UI 要求：

- 生态名称输入框必须短而明确。
- 源质印记使用图标、等阶、小标题和一句短描述。
- 玩家只能选择一个印记。
- 创建完成后，游客 id 和存档 id 保存在浏览器本地。
- 不在第一版做注册、密码、手机号等正式账号流程。

## 4. 主操作

第一版主操作为：

```text
催化潮池
```

主操作效果：

- 立即获得少量有机质、能量或矿物质。
- 轻微改变潮池状态。
- 推进当前目标进度。
- 触发短反馈文案。
- 有概率积累突变倾向。

主操作不能失败。  
如果操作需要资源消耗，它不应该是开局主操作，而应放在后续策略操作中。

## 5. 引导阶段

第一版采用分阶段引导。

### 阶段 1：照看潮池

玩家只看到：

- 始源潮池
- 当前目标：富集有机质
- 主操作：催化潮池
- 目标进度：有机质达到 20

解锁条件：

```text
有机质 >= 20
```

解锁内容：

- 演化入口
- 第一个演化节点“有机富集”

### 阶段 2：点亮结构

玩家看到：

- 潮池
- 当前目标：点亮有机富集
- 演化入口

完成第一个节点后：

- 高级操作逐步出现
- 目标切换为“推动自复制链”
- 资源进度扩展为有机质、矿物质、稳定性

### 阶段 3：等待谱系

解锁自复制链后，玩家目标变为：

```text
制造足够的突变与稳定窗口，等待第一条谱系出现
```

解锁内容：

- 图鉴入口在第一个物种出现后显示。
- 最近发现卡片显示在首屏，而不是让玩家必须进图鉴看。

### 阶段 4：生态筛选

当出现物种、稳定性下降或事件触发后：

- 玩家开始看到生态压力。
- 灾变、濒危、化石遗产等系统逐步出现。
- 遗产入口只在第一次遗产形成后出现。

## 6. 目标卡

目标卡是玩家的主要引导。

目标卡包含：

- 标题
- 一句话解释
- 进度条
- 下一步按钮或提示

目标示例：

```text
富集有机质
潮池需要足够的分子材料，才可能留下第一批结构痕迹。
进度：12 / 20 有机质
主操作：催化潮池
```

目标卡不能写成长教程。  
每次只告诉玩家一个行动。

## 7. 资源呈现

资源不应开局全量展示。

开局只显示：

- 有机质
- 能量

第一个节点后显示：

- 矿物质
- 稳定性

自复制链后显示：

- 突变点
- 生物量

资源展示应以“目标进度”和“短数字”为主。

## 8. 底部入口

底部入口必须渐进解锁。

```text
开局：潮池
有机质接近目标：潮池 / 演化
发现物种后：潮池 / 演化 / 图鉴
产生遗产后：潮池 / 演化 / 图鉴 / 遗产
经历多个事件后：增加日志入口
```

底部入口是功能切换，不是系统展示墙。

底部入口表现形式：

- 使用图标作为第一识别信号。
- 小字只保留 2 到 4 个字，例如“潮池”“演化”“图鉴”。
- 不使用长按钮文案。
- 当前入口用颜色、亮度或轻微高亮标识。

## 8.1 首次引导规则

任何第一次出现的功能，都必须有上手引导。

需要引导的场景：

- 第一次进入游戏。
- 第一次创建生态。
- 第一次看到潮池首页。
- 第一次解锁底部入口。
- 第一次进入演化、图鉴、遗产、日志、设置等页面。
- 第一次使用策略操作。
- 第一次触发生态事件。
- 第一次获得新资源、新参数或新状态。

引导形式：

- 优先使用半屏短提示、目标卡、解锁弹层或空状态引导。
- 每次只解释当前玩家马上要用的一个功能。
- 文案控制在 1 到 3 句，不写成长教程。
- 引导必须给出明确下一步，例如“点亮”“收录”“选择”“返回潮池”。

禁止：

- 新入口静默出现。
- 第一次进入页面没有说明这个页面能做什么。
- 一次性解释多个未来系统。
- 用长篇说明代替可操作引导。

## 9. 操作类型

操作分为三层。

### 主操作

始终在首屏最明显位置。

示例：

- 催化潮池
- 引导闪光
- 促进分化

表现形式应是大图标按钮加短标签，例如：

```text
[闪光图标]
催化
```

### 策略操作

在完成初期引导后出现。

示例：

- 潮汐扰动
- 提高温度
- 投放矿物
- 稳定膜泡

策略操作可以有资源消耗和长期代价。

### 系统操作

放在二级页面。

示例：

- 解锁演化节点
- 查看图鉴详情
- 查看化石遗产
- 查看日志

## 10. 反馈设计

每次有效操作都需要至少一种反馈：

- 数字进度变化
- 潮池动画变化
- 一句短反馈
- 新目标出现
- 新入口解锁
- 新物种卡片浮现

反馈文案示例：

```text
矿物晶面捕获了一批有机分子。
浅水升温，反应变得更活跃。
薄膜结构在潮汐边缘短暂成形。
一条异常链体留下了复制痕迹。
```

## 11. 动态状态显示

放置游戏的首屏必须持续变化，不能只在玩家手动点击后更新。

显示策略：

- 前端每秒进行本地预测 tick，让资源、目标进度和可见状态持续变化。
- 后端仍然是权威存档，每隔一段时间同步一次。
- 本地预测只用于展示，不直接写入存档。
- 玩家执行关键操作后，立即请求后端并用权威状态校准页面。
- 如果前端预测和后端返回有差异，以后端为准。

## 12. 第一版手机端验收标准

- 玩家开局 3 秒内知道要点击哪里。
- 开局第一下点击必须有效。
- 首屏不出现完整资源表和完整系统入口。
- 30 秒内至少出现一次明确目标进展。
- 不点击时，资源或目标进度也应持续变化。
- 2 到 3 分钟内解锁第一个新入口。
- 新入口出现时有文字反馈。
- 演化、图鉴、遗产、日志都不是开局全量显示。
- 界面看起来像游戏主界面，而不是控制台。

## 13. 禁止事项

- 禁止把所有系统同时铺开。
- 禁止开局展示完整参数表。
- 禁止让玩家第一步就遇到资源不足。
- 禁止用长说明替代引导。
- 禁止把主操作藏在面板里。
- 禁止让底部导航一开始就显示所有功能。

## 14. 当前前端落地约定

2026-05-09 的第一轮移动端 UI 重构按以下约定实现：

- 首页只承载当前目标、潮池主场景、主操作、少量资源和最近反馈。
- 生态干预使用底部半屏选择，而不是长期占据首页。
- 设置页作为测试与存档管理入口常驻，包含本地存档重置。
- 异常、空状态、系统解锁、首次引导都必须保持游戏世界感。
- 新入口出现时优先用奖励弹层承接，再允许玩家进入对应系统。
- 后续新增页面必须先补到 `docs/mobile-ui-prototypes.md` 或本文件，再进入实现。

## 16. 拖拽元素入池 — 核心操作设计

第一版主操作不使用传统点击按钮。玩家通过 **拖拽潮池边缘的浮游元素进入池中心** 来触发催化反应。

### 16.1 元素类型

池子周围随机漂浮 4 种元素，拖入池中心触发对应反应：

| 元素 | 视觉 | 对应操作 | 正向效果 (70%) | 负向效果 (25%) |
|---|---|---|---|---|
| 矿物晶体 | 青绿色多面体 | 矿物沉积 | 矿物质+，稳定性+ | 沉积过快，有机质产出短暂减速 |
| 能量闪光 | 金色闪电球 | 增强光照 | 能量+，突变点+ | 能量冲击，挥发性抬高 |
| 有机液滴 | 蓝绿色液泡 | 潮汐扰动 | 有机质+，能量+ | 轻微紊乱，稳定性小降 |
| 异常脉冲 | 紫色光晕（稀有） | 高温催化 | 大量突变+，稀有物种概率 | 永不为负 |

### 16.2 随机性

- 70% 正向反馈：正常生效，绿色浮字显示收益
- 25% 负向反馈：仍执行操作，但红色提示附带轻微副作用，带屏幕抖动
- 5% 稀有脉冲：金色光效，多资源同时获得

### 16.3 生成规则

- 同时最多 4 个元素可见
- 每 4~10 秒在池子边缘随机位置生成一个新元素
- 达到上限时，最旧的元素缓慢淡出消失
- 元素在池边缓慢浮动绕行，带轻微呼吸缩放

### 16.4 拖拽交互

- 按住元素开始拖拽，拖尾粒子跟随
- 拖向池心（半径 60px 内）松手 → 被吸收，触发反应
- 松在池外 → 元素弹回原位
- 拖拽时池面出现微弱引力光晕

### 16.5 引导

- 首次进入潮池 → 第一个元素 3 秒后出现，伴随从元素指向池心的流动箭头 + 文字「把它引入潮池」
- 首次拖拽成功 → 解锁提示「潮池吸收了新的反应材料」
- 首次负反馈 → 短提示「反应偶尔失衡，生态自会调整」
- 第 3 次拖拽后 → 引导箭头和提示永久消失

### 16.6 禁止

- 不放多个元素类型让玩家选（已通过随机生成和上限控制）
- 不让所有元素都是正向（25% 负向保持紧张感）
- 不让元素一次太多（上限 4 个避免杂乱）
- 不让操作失败（松手在池外只弹回，不惩罚）
