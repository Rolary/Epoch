# 《生态纪元》游戏设计总纲

本文档是后续产品与开发调整的最高层设计约束。任何新系统、页面、数值或 AI 生成能力，都必须服务于这里定义的体验方向。

## 1. 主概念

```text
从潮池到群星，见证一颗星球自己的生命史。
```

游戏覆盖从生命起源到最终文明与星际生态的演化过程。第一版只聚焦“生命诞生篇”，但所有设计都要为长期主线预留连续成长空间。

## 2. 玩家身份

玩家不是全知上帝，也不是文明领袖。

玩家更像接入了一个“生态纪元系统”的观察者与引导者：

- 不能直接创造指定物种。
- 可以调节环境、施加选择压力、选择长期倾向。
- 可以影响某类生命更容易出现，但不能完全决定结果。
- 需要接受繁盛、分化、失衡、灭绝和遗产共同构成生命史。

## 3. 核心体验

玩家打开游戏后的核心感受应该是：

```text
我正在照看一个活着的始源潮池。
我的每次选择都会改变这颗星球的生命史。
生命不是凭空出现的，而是一步步从材料、结构、复制、细胞走向复杂生态。
```

## 4. 第一版定位

第一版主题：

```text
生命诞生篇：让第一批生命从始源潮池中出现。
```

第一版不是完整多纪元游戏，也不是系统功能展示页。第一版必须把早期线性主线做清楚，让玩家能理解生命从无到有的过程。

第一版阶段：

```text
富集材料
-> 留下痕迹
-> 学会复制
-> 包成细胞
-> 追逐光照
```

## 5. 不可违背规则

- 不做轮回重置。
- 不清空玩家已有生命史。
- 不把灭绝做成单纯失败。
- 不让 AI 决定核心数值、价格、概率或解锁条件。
- 不把首页做成资源控制面板。
- 不在开局展示完整资源表、完整环境参数、完整演化树和完整导航。
- 不让玩家第一步遇到资源不足。
- 不把图鉴做成纯文本收藏，物种必须尽量有生态功能。
- 不用单一资源门槛推进纪元，关键推进应包含节点、物种、事件或生态状态。
- 不新增脱离“生命史推进”的独立系统。
- 不允许新功能无引导出现，包括第一次进入游戏、首次解锁页面、首次触发事件和首次获得新资源。

## 6. 页面判断标准

任何页面都要先回答一个问题：

```text
这个页面帮助玩家推进当前生命史了吗？
```

如果答案是否定的，这个页面不应进入第一版。

首页必须满足：

- 一个当前目标。
- 一个活着的潮池主场景。
- 一个主要操作。
- 少量关键资源。
- 最近一次反馈。
- 只展示已解锁入口。

首页禁止：

- 全量资源列表。
- 全量环境参数。
- 多个并列系统面板。
- 后台仪表盘式布局。
- 开局直接铺满所有功能入口。

## 7. 系统解锁原则

系统必须随着主线逐步出现，而不是一次性展示。

推荐节奏：

```text
创建生态：生态名称、源质印记。
富集材料：只展示潮池、目标、主操作、少量资源。
接近首次节点：解锁演化入口。
出现第一条谱系：解锁图鉴入口。
发生筛选或灭绝：解锁化石遗产入口。
经历多个事件：解锁演化日志入口。
```

每次解锁新系统，都要通过目标卡、事件弹层、奖励弹层或日志提示承接，不能静默增加按钮。

## 8. 引导原则

任何玩家第一次遇到的内容，都必须有短引导帮助上手。

必须提供引导的场景：

- 第一次进入游戏。
- 第一次创建生态。
- 第一次看到潮池首页。
- 第一次解锁演化入口。
- 第一次进入演化页。
- 第一次使用策略操作。
- 第一次触发生态事件。
- 第一次发现物种。
- 第一次进入图鉴。
- 第一次形成化石遗产。
- 第一次进入遗产页。
- 第一次查看演化日志。
- 第一次获得新资源或新生态参数。

引导形式应短、轻、游戏化：

- 目标卡一句话提示。
- 半屏首次引导弹层。
- 系统解锁奖励弹层。
- 生态事件短说明。
- 页面空状态中的下一步提示。

引导禁止：

- 不写成长教程。
- 不一次解释多个系统。
- 不阻断玩家连续操作太久。
- 不使用后台说明书式文案。
- 不让新按钮或新页面静默出现。

## 9. AI 使用边界

AI 应负责让生命史独一无二：

- 物种命名。
- 物种短描述。
- 外观提示词。
- 习性标签。
- 谱系摘要。
- 演化日志。
- 事件叙事包装。
- 生命史报告。

AI 不负责：

- 资源公式。
- 节点价格。
- 事件概率。
- 惩罚强度。
- 主线解锁条件。
- 关键数值平衡。

系统必须先生成结构化生态事实，再让 AI 生成表达。

## 10. 成功标准

第一版成功不是功能多，而是玩家能在 10 到 20 分钟内明确感受到：

```text
我从一片潮池开始，推动材料富集，见证结构稳定、复制出现、细胞诞生，并记录下属于这颗星球的第一段生命史。
```

如果玩家只觉得自己在点资源按钮、看面板涨数值，则设计失败。
