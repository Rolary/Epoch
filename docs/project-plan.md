# 《生态纪元》项目方案

## 1. 项目定位

后续开发优先遵守以下约束文档：

```text
docs/game-design-bible.md
docs/main-progression.md
docs/first-session-flow.md
docs/mobile-gameplay-ux.md
```

其中 `docs/game-design-bible.md` 是最高层设计约束，用于避免游戏被实现成控制面板式系统堆叠。

《生态纪元》是一款网页端放置演化游戏。

项目核心目标是做出接近《物种起源》那种持续解锁、数值增长、演化爆发的爽感，但避免传统放置游戏常见的“轮回重置”路线。

游戏不从“大爆炸”开始，而是从一颗年轻行星上的 **始源潮池** 开始。玩家通过环境调控、演化压力和随机事件，引导生命从早期分子结构逐步发展为复杂生态、智慧物种和星际生态。

核心卖点：

- 不轮回，不清空，生命史连续推进
- 每个物种都会进入图鉴，每次灭绝都会留下遗产
- AI 生成独一无二的物种、外观、习性、谱系与演化日志
- 固定主线保证节奏，随机事件和 AI 内容保证差异化

一句话概念：

> 从始源潮池开始，引导生命在随机事件与 AI 生成内容中不断分化、繁盛、灭绝，并沉淀出只属于玩家的生态纪元。

主概念表达：

> 从潮池到群星，见证一颗星球自己的生命史。

这个表达用于对外说明项目跨度：游戏不是只做早期微生物，也不是单纯经营文明，而是从生命起源一路推进到最终文明与星际生态。

## 2. 背景设定

海洋刚刚退去，留下了一片温热的始源潮池。

闪电划过年轻的大气，火山灰落入浅水，矿物晶体在一次次蒸发与回潮中沉积。无数分子在这里碰撞、断裂、重组，其中极少数开始留下自己的痕迹。

玩家接入了名为“生态纪元”的系统。

这里没有固定的生命图鉴，也没有完全相同的演化路线。AI 会根据玩家的环境选择、随机事件和突变轨迹，生成只属于当前星球的物种名称、外观、习性、谱系与演化日志。

玩家不能直接创造生命，只能调节环境，施加压力，等待不可预测的分支出现。

世界观关键词：

- 年轻行星
- 始源潮池
- 连续生命史
- 环境调控
- 选择压力
- AI 图鉴
- 化石遗产
- 独一无二的演化档案

## 3. 起点阶段

起点阶段确定为：

> 始源潮池

选择理由：

- 比“原始汤”更有画面感和质感
- 保留自然生命起源的直觉
- 适合放置游戏早期从分子、有机质、能量、稳定性开始增长
- 适合解释潮汐、蒸发、闪电、火山灰、矿物晶面等随机事件

早期环境元素：

- 温热浅水
- 潮汐涨落
- 闪电
- 火山灰
- 矿物晶体
- 有机分子
- 蒸发与回潮循环

## 4. 主线纪元

主线采用连续推进，而不是重置轮回。

长期章节建议使用更容易理解的篇章表达：

```text
生命诞生篇：从潮池中诞生第一批生命
细胞时代篇：原初细胞建立最早生态
蓝绿黎明篇：光合作用改变星球大气
海洋繁盛篇：复杂生命在海中爆发
登陆扩张篇：生命第一次走向陆地
森林巨兽篇：陆地生态形成繁盛时代
智慧火种篇：智慧物种开始使用工具与语言
文明生态篇：文明成为新的生态力量
群星播种篇：生命走出母星，进入星际生态
```

对应底层演化阶段：

```text
始源潮池
-> 自复制分子
-> 原初细胞
-> 光合菌群
-> 氧化纪元
-> 真核跃迁
-> 多细胞生命
-> 海洋爆发
-> 陆地扩张
-> 森林纪元
-> 巨兽时代
-> 智慧萌芽
-> 文明生态
-> 星际播种
```

每个阶段应该带来：

- 新资源
- 新演化节点
- 新生态位
- 新随机事件池
- 新 AI 图鉴内容
- 新物种遗产类型

## 5. 核心循环

目标循环：

```text
环境产出资源
-> 玩家调控环境
-> 解锁生命结构
-> 触发生态爆发
-> 生成随机物种
-> 写入 AI 图鉴
-> 物种影响资源产出
-> 随机事件筛选生态
-> 灭绝物种沉淀为遗产
-> 推进下一个纪元
```

避免传统循环：

```text
攒资源 -> 重置 -> 获得倍率 -> 重新再来
```

采用连续成长：

```text
攒资源 -> 解锁复杂度 -> 旧系统沉淀为历史资产 -> 开启新生态位和新纪元
```

## 6. 放置爽感设计

游戏需要保留“停不下来”的放置体验。

爽感来源不依赖轮回，而来自以下机制：

- 连续解锁
- 连锁爆发
- 多层进度条
- 演化树点亮
- 随机物种发现
- AI 图鉴新增
- 事件触发分化
- 灭绝转化为遗产

关键设计原则：

- 每隔几十秒到几分钟给玩家一个新变化
- 一个重要节点应引发多个系统变化
- 不只是 `+10%`，而是解锁资源、分支、事件和图鉴内容
- 玩家始终能看到“差一点就到了”的目标

示例：

```text
解锁光合作用
-> 能量产出暴涨
-> 氧气开始积累
-> 蓝藻类物种出现
-> 好氧代谢分支解锁
-> 厌氧生命受到压力
-> 氧气事件进度条出现
-> AI 图鉴新增多个生命记录
```

## 7. 资源体系

早期资源：

- 有机质
- 能量
- 矿物质
- 稳定性
- 突变点
- 生物量

中后期可扩展资源：

- 氧气
- 适应力
- 基因多样性
- 生态复杂度
- 生态位
- 捕食压力
- 共生度
- 智慧火花
- 文明影响
- 星球适应性

资源设计原则：

- 早期资源少，反馈清晰
- 每个新纪元增加少量新资源，而不是一次性膨胀
- 资源之间要有生态关系，例如光合作用提高能量和生物量，同时引入氧气压力

## 8. 玩家操作

玩家不是直接创造物种，而是通过环境和压力间接引导演化。

早期操作：

- 增强光照
- 矿物沉积
- 潮汐扰动
- 提高温度
- 增加有机质浓度
- 释放火山灰

中后期操作：

- 调节氧气
- 扩张浅海
- 制造捕食压力
- 促进共生
- 保护濒危谱系
- 保存化石遗产
- 开启新生态位

操作设计原则：

- 操作要制造短期收益和长期演化影响
- 操作可以提高某类物种或特性的出现概率
- 操作不应完全决定结果，随机性和 AI 生成要保留惊喜

## 9. 演化树

演化树是核心成长界面。

节点不是单纯数值升级，而应代表生命结构、生态能力或重大跃迁。

早期节点示例：

```text
有机富集
自复制链
原始膜泡
代谢回路
原初细胞
光合作用
早期共生
氧化适应
```

节点效果可以包括：

- 提高资源产出
- 解锁新资源
- 解锁新事件
- 提高 AI 分化概率
- 解锁新物种模板
- 推进纪元进度

## 10. 随机事件

随机事件不是单纯奖励或惩罚，而是推动演化分叉。

事件类型：

- 环境事件
- 生物事件
- 灾变事件
- 发现事件

早期事件示例：

```text
闪电富集：提高能量和高能反应概率
火山灰沉降：增加矿物质和催化表面
蒸发窗口：提高有机质浓度，但降低稳定性
潮池干涸：筛选现存物种，可能形成化石遗产
异常分支：触发 AI 生成物种
```

中后期事件示例：

```text
氧气危机
寒潮纪元
浅海扩张
捕食压力上升
共生机会
寄生爆发
化石出土
异常基因
```

灾变设计原则：

- 不清空玩家成果
- 通过筛选改变生态结构
- 让部分物种灭绝或化石化
- 形成永久遗产或新的演化压力

## 11. AI 系统

AI 是本项目的重要差异化点。

AI 不只是生成几句描述，而是游戏的“生命史记录系统”。

AI 负责：

- 物种命名
- 物种描述
- 物种外观或插画提示词
- 习性标签
- 谱系记录
- 演化日志
- 生态摘要
- 随机事件文本包装
- 图鉴内容个性化

AI 不负责：

- 核心数值
- 升级价格
- 概率权重
- 惩罚强度
- 主线解锁条件

原因：

放置游戏数值必须稳定。AI 可以让世界变丰富，但底层规则、平衡和解锁条件应由系统控制。

推荐生成流程：

```text
系统先生成结构化数据
-> AI 根据数据生成名称、描述、外观、日志
-> 系统保存到图鉴
-> 玩家获得可见奖励和生态影响
```

结构化物种数据示例：

```text
类型 = 光合原核生命
生态位 = 浅海表层
特性 = 群体漂浮、薄膜结构、高效光合
数值模板 = 能量产出型
稀有度 = 普通
时代 = 始源潮池
```

AI 输出示例：

```text
名称 = 蓝膜浮群
描述 = 蓝膜浮群出现在浅海表层，依靠群体漂浮和薄膜结构捕获光照。它们改变了潮池中的能量流动，是当前星球最早的光合谱系之一。
插画提示词 = 科学图鉴插画风格，浅海潮池中的蓝绿色薄膜状微生物群，微弱荧光，矿物晶体背景
```

## 12. AI 随机性和独一无二

需要在产品体验上明确传达：

- 没有固定生命图鉴
- 没有完全相同的演化路线
- 玩家操作、随机事件和突变轨迹会共同影响 AI 生成内容
- 每个玩家都会获得不同的物种、谱系、日志和化石遗产

开场文案已明确体现：

> 这里没有固定的生命图鉴，也没有完全相同的演化路线。AI 会根据你的环境选择、随机事件和突变轨迹，生成只属于你的物种名称、外观、习性、谱系与演化日志。

## 13. 图鉴系统

图鉴不是收藏展示，而是长期成长系统。

每个物种卡包含：

- 名称
- 插画
- 生命类型
- 生态位
- 状态
- 产出
- 特性
- 生态作用
- 发现纪元
- 祖先关系
- 灭绝后遗产
- AI 生成描述

物种状态：

```text
未发现
现存
繁盛
濒危
灭绝
化石化
祖先节点
异常种
```

核心原则：

> 灭绝不是失败，而是历史资产。

遗产示例：

```text
早期鱼类化石：脊椎类演化速度 +10%
巨型蕨类遗产：陆地生态承载量 +8%
古菌谱系：极端环境生命产出 +12%
```

## 14. 化石遗产

化石遗产用于替代传统重置倍率。

机制目标：

- 让物种灭绝也有价值
- 保留连续生命史感觉
- 提供长期成长
- 避免“全部清空重来”

流程：

```text
随机事件或生态压力发生
-> 部分物种退出现存生态
-> 物种变为化石化或灭绝
-> 形成遗产
-> 遗产提供持续加成或降低后续成本
```

## 15. UI 与体验方向

首屏不做营销落地页，而是直接进入游戏体验。

当前产品优先面向手机端体验。界面应按移动端放置游戏来组织，而不是桌面后台面板。

手机端操作与引导的详细规范见：

```text
docs/mobile-gameplay-ux.md
```

当前设计结构：

- 顶部：当前纪元、同步状态、星球性格摘要
- 首屏核心：始源潮池生态视窗
- 资源摘要：少量关键资源以紧凑宫格展示
- 环境调控：使用适合拇指点击的按钮
- 底部导航：演化树、AI 图鉴、化石遗产、演化日志
- 二级内容：通过底部 Tab 切换，不在首屏一次性铺满

视觉方向：

- 自然演化
- 轻科幻系统感
- 潮池、矿物、浅海、闪电
- 图鉴和日志偏科学档案风格

移动端 UI 原则：

- 单手可操作，主要按钮不小于 44px 高。
- 底部导航固定，但只展示已解锁功能，不能一开始铺满所有系统。
- 首屏先呈现“活着的潮池”，再呈现系统面板。
- 首屏必须有明确的当前目标、主操作按钮和即时反馈，不能像控制面板。
- 信息分层，不同时展示过多面板。
- 资源数字必须短、清晰、可扫读。
- 面板不做桌面仪表盘式密集排布。

渐进式放出原则：

- 开局只展示潮池、当前目标、主操作和少量关键资源。
- 演化树在资源接近首次解锁时出现。
- 图鉴在发现第一个物种后出现。
- 化石遗产在第一次灭绝或化石化后出现。
- 日志在玩家经历多个事件后出现。
- 高级环境参数在玩家至少完成一个演化节点后出现。
- 每次新系统出现都应通过目标卡或日志给出简短反馈，而不是无声增加按钮。

## 16. 玩家决策张力

游戏不能只让玩家点击收益最高的按钮，而应让玩家持续面对生态取舍。

核心张力：

- 稳定性 vs 突变率：稳定性高，资源增长更稳；突变率高，新物种和异常分支更多，但灭绝和失衡风险上升。
- 生物量 vs 多样性：单一优势谱系产出更高，但生态更脆弱；多样性提高抗灾能力和分支概率。
- 短期富集 vs 长期压力：闪电、火山灰、强光照等操作带来短期收益，同时可能积累温度、毒性、氧化或干涸压力。
- 保护谱系 vs 释放生态位：保护濒危物种可延续现有能力，让其退出生态也可能形成遗产或释放新生态位。
- 事件控制 vs 随机惊喜：玩家可以提高某类事件或物种出现概率，但不能完全指定结果。

目标体验：

```text
玩家选择一种环境倾向
-> 资源结构发生变化
-> 某些谱系更容易繁盛
-> 某些谱系承受压力
-> 事件筛选生态
-> 图鉴和遗产记录这段历史
```

判断标准：

> 玩家每隔几十秒到几分钟，不只是领取奖励，而是要做一个会影响星球性格和生命史走向的选择。

## 17. 物种生态功能模型

图鉴中的物种应当是生态系统的一部分，而不是纯收集卡。

每个物种至少应包含：

- 生态角色：生产者、分解者、捕食者、共生者、寄生者、工程物种、极端环境适应者等。
- 生态位：潮池表层、矿物晶面、浅海底部、热泉边缘、薄膜群落、沉积层等。
- 生态影响：资源产出、稳定性、多样性、事件概率、其他物种状态。
- 脆弱点：怕氧气、怕寒冷、怕干涸、怕捕食、依赖共生对象等。
- 谱系关系：祖先、后代、旁支、异常分支。

示例：

```text
名称：蓝膜浮群
生态角色：初级生产者
生态位：浅海表层
正面影响：能量产出提高，有机质积累提高
副作用：氧气压力缓慢上升
脆弱点：潮池干涸、强烈矿物沉降
演化倾向：提高光合类分支和好氧适应节点出现概率
```

这样玩家获得新物种时，不只是图鉴多一条记录，而是生态系统多了一个会改变局势的组成部分。

## 18. 灭绝遗产分类

化石遗产用于替代传统重置倍率，但不应只做成简单的百分比加成。

建议遗产类型：

- 化石遗产：提供稳定加成，例如某类演化成本降低、某类资源产出提高。
- 祖先遗产：解锁新分支或提高关键结构出现概率。
- 生态空位：物种退出后释放生态位，引发后续物种爆发。
- 警示遗产：降低类似灾变的损失，或提高对应适应力。
- 博物档案：主要用于生命史报告、图鉴导出和叙事价值。

示例：

```text
潮隙链体化石
类型：祖先遗产
效果：自复制结构相关节点成本 -8%
叙事：这组链体没有延续到后世，却把复制错误变成了演化材料。
```

核心原则：

> 灭绝不是失败，也不只是奖励，而是历史对未来生态方向的重塑。

## 19. 纪元推进条件

纪元推进不应只依赖资源门槛。

推荐使用多条件推进：

```text
进入“原初细胞”纪元需要：
- 解锁自复制链
- 解锁原始膜泡
- 稳定性达到指定水平
- 至少发现 2 个分子类谱系
- 触发过 1 次异常分支或筛选事件
```

纪元推进条件应包含：

- 资源条件
- 演化节点条件
- 物种或谱系条件
- 事件经历条件
- 生态状态条件

这样纪元变化更像生态跃迁，而不是普通升级按钮。

## 20. AI 生成约束

AI 生成应保持丰富，但必须被结构化数据约束。

系统先确定：

```text
时代
生命类型
生态角色
生态位
特性
稀有度
祖先关系
数值模板
事件背景
```

AI 再输出：

```text
名称
短描述
外观提示词
习性标签
谱系摘要
演化日志文本
灭绝或遗产提示
```

名称风格应随纪元演进：

- 早期：蓝膜浮群、灰晶链体、潮隙原胞、铁纹薄膜菌。
- 中期：棘鳍滤食群、陆隙孢林、环礁共生体。
- 后期：晨岸智族、星潮播种体、轨道生态群。

AI 的目标不是即兴发挥，而是在规则确定的生态事实之上，生成独一无二的表达。

## 21. 星球性格系统

玩家长期选择应逐渐塑造星球性格。

隐藏或半公开倾向示例：

- 高突变星球：异常分支多，灭绝也更频繁。
- 稳定富氧星球：生产效率高，但厌氧古老谱系压力大。
- 灾变频发星球：筛选事件多，化石遗产更丰富。
- 共生繁盛星球：物种依赖关系强，生态复杂度提升快。
- 捕食压力星球：谱系竞争强，适应力和防御结构更容易出现。
- 极端环境星球：极端生态位更容易形成，常规生物量增长较慢。

星球性格可影响：

- 事件池权重
- 物种生成模板
- AI 文案风格
- 图鉴统计
- 生命史报告
- 后续纪元的分支倾向

它的作用是强化“这颗星球是我的选择和随机历史共同塑造的”。

## 22. 第一版范围建议

第一版不追求多纪元完整流程，而应把早期闭环做完整。

第一版主线和首次体验的执行细节见：

```text
docs/main-progression.md
docs/first-session-flow.md
```

第一版聚焦“生命诞生篇”，目标是让玩家亲手见证第一批生命如何从始源潮池中出现。

面向玩家的阶段表达：

```text
富集材料：潮池里开始积累有机质、能量和矿物。
留下痕迹：一些分子结构开始稳定存在。
学会复制：出现能自我复制的早期链体。
包成细胞：膜泡把反应包裹起来，形成原初细胞。
追逐光照：生命开始走向光合作用，新的生态爆发即将到来。
```

对应底层范围：

```text
始源潮池
-> 自复制分子
-> 原初细胞
-> 光合作用前夜
```

内容规模建议：

- 6 种以内早期资源。
- 12 到 20 个演化节点。
- 15 到 25 个随机事件。
- 20 个左右可生成物种模板。
- 3 到 5 种灭绝或化石遗产。
- 1 条明确纪元推进线。
- 至少 1 次生态爆发体验。

第一版必须验证的核心链路：

```text
我的选择
-> 环境变化
-> 物种出现
-> 生态改变
-> 灾变筛选
-> 遗产沉淀
-> 新分支出现
```

第一版成功标准不是内容多，而是玩家能感受到自己正在塑造一段不可复刻的生命史。

## 23. 生态命名与源质印记

开局不应直接进入匿名存档，而应让玩家创建自己的生态。

第一版创建流程：

```text
输入生态名称
-> 随机出现 3 个源质印记
-> 玩家选择 1 个
-> 创建后端存档
-> 浏览器本地保存游客 id 和存档 id
-> 进入始源潮池
```

生态名称相当于玩家当前星球/生态档案的身份。第一版不做正式账号体系，游客 id 和存档 id 保存在浏览器本地。

### 源质印记

天赋系统命名为：

```text
源质印记
```

含义：

> 在生命史尚未展开前，潮池中已经存在某种长期倾向。它不是直接创造生命，而是塑造生命更容易走向的方向。

源质印记拥有等阶：

- I 阶：微弱倾向，开局和早期节点主要获得。
- II 阶：稳定倾向，中期纪元跃迁时出现。
- III 阶：星球级倾向，重大生态跃迁或后期纪元出现。

第一版先实现 I 阶。

开局源质印记示例：

```text
矿晶温床：矿物质获取提高，早期结构更容易稳定。
闪电亲和：能量获取提高，突变点缓慢增加。
潮汐记忆：有机质获取提高，潮汐扰动代价降低。
稳膜倾向：稳定性恢复提高，灾变筛选压力降低。
异变火花：突变点获取提高，更容易出现异常谱系。
```

### 获取方式

- 第一次创建生态时，随机 3 个源质印记，玩家选择 1 个。
- 每次完成重大演化节点或纪元跃迁时，再随机 3 个源质印记，玩家选择 1 个。
- 已选择的源质印记永久生效，贯穿整个存档。

重大节点第一版定义：

- 自复制链
- 原初细胞
- 感光色素

### 设计原则

- 源质印记只能影响倾向、产出、代价或概率，不能替玩家直接解锁主线。
- 源质印记应强化“我的生态和别人不同”的感觉。
- 选择应简单清楚，使用图标、等阶和短描述，而不是长说明。
- 每次获得新印记都要有明确反馈，像一次小型奖励选择。

## 26. 实现注意事项

跨 agent 继续开发时应遵守：

- 不要把游戏改成轮回重置玩法
- 不要让 AI 直接控制核心数值平衡
- 不要删除始源潮池起点设定
- 不要把图鉴做成纯装饰，应保留成长价值
- 灾变事件不应清空成果，应形成筛选和遗产
- 固定主线和随机分支要并存
- 新功能优先围绕“连续生命史”和“独一无二图鉴”展开
- 玩家操作必须保留短期收益和长期代价
- 物种必须尽量拥有生态功能，不应只是文本收藏
- 纪元推进应优先使用多条件生态跃迁，而不是单一资源门槛
- 源质印记是永久成长系统，不能做成一次性临时 buff

核心判断标准：

> 玩家是否感觉自己正在见证一部不可复刻的生命史，而不是反复刷同一套倍率。
